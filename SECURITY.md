# Security Policy

We take the security of this project seriously. Thank you for helping keep the community safe.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

To report a security vulnerability:

1. **Use GitHub Private Vulnerability Reporting** (recommended): If this repository has [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability) enabled, open the **Security** tab, then **Advisories**, and click **Report a vulnerability**.
2. **Otherwise**, open a private issue or contact the maintainers listed in the repository description.

Please include as much of the following as you can:

- The type of issue (e.g., credential exposure, injection, XSS)
- Full paths of source file(s) related to the issue
- The location of the affected source code (tag/branch/commit or URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

We will acknowledge your report and work with you on a coordinated disclosure timeline.

## Supported Versions

Security updates are applied to the current stable release. We encourage users to stay on the latest release.

## Policy

We follow coordinated disclosure: report privately, allow maintainers time to address the issue, and disclose publicly only after a fix is available or after an agreed timeline.
