# Data Directory (`data/`) - Lowest-Level Best Practices & Checklist

This document contains the lowest-level, file-by-file checklist for every component within the `data/` directory.

## Directory Overview
The `data/` directory holds raw data exports, CSVs, and state configurations required for or generated by the application workflows.

---

## File-Level Checklists

### 1. `[platform]_trending.csv` (e.g., `youtube_trending.csv`, `instagram_trending.csv`)
- **Feature**: Exported outputs of the `trending_videos` submodule.
- **Detailed Checklist**:
  - [x] **Data Integrity Protocol**: Ensure all CSV columns exist (`Platform`, `Title`, `URL`, `Views`, `Category`, `VideoType`, `PublishedDate`) uniformly across files.
  - [x] Handle data sanitization (e.g., escaping commas inside strings in the CSV writer module).

### 2. Configuration & State Artifacts
- **Feature**: Any temporary or persisted workflow states generated during runtime.
- **Detailed Checklist**:
  - [x] **Data Privacy Constraint**: Ensure `.gitignore` policies explicitly ignore environments or configurations holding PII (Personally Identifiable Information) or passwords.
  - [x] Data formats should prioritize universal serializability (`JSON` or `YAML`).

*(Note: Data extraction directories are dynamic, so guidelines cover schemas and processing rather than hardcoded file tracking.)*
